# 完善 trading-toolkit-web 网页版功能计划

## 概述

参考同目录微信小程序项目 `trading-toolkit-mp`，为 Vue3 网页版 `trading-toolkit-web` 补齐缺失的核心功能。网页版技术栈为 Vue3 + Vite + Pinia + VueRouter + ElementPlus + SCSS。

## 现状分析

### 网页版现有功能
- 首页（Home）：市场概览卡片、硬编码名言轮播、日历、情绪指标
- 可转债（Convertible）+ 转债详情（BondDetail）
- LOF 基金（Lof）
- 港股打新（Hkipo）+ 港股打新详情（HkipoDetail）
- 自选管理（Favorites）
- 设置（Settings）：仅暗黑模式开关 + 缓存清除

### 小程序有但网页版缺失的功能
1. **API 日志页面**：查看后端接口调用日志、搜索、清空、连通性检测
2. **名言管理页面**：管理首页轮播名言的增删改、恢复默认
3. **设置页面增强**：云托管地址配置、关于信息、开发工具入口
4. **首页名言动态化**：目前硬编码 4 条名言，需改为从 localStorage 动态读取

## 具体变更

| `src/views/Home.vue` | 修改 | `quotes` 从硬编码改为 `quoteManager.getQuotes()` 读取；无数据时回退到默认值 |
| `src/components/QuoteCarousel.vue` | 修改 | 模板中 `q.source` 改为 `q.author`，保持现有跑马灯动画逻辑 |

**技术决策**：名言字段统一为 `author`，与小程序保持一致，替代网页版当前使用的 `source`。

### 三、API 日志页面（ApiLog.vue）

**目标**：实现后端接口调用日志的查看、搜索、清空与连通性检测。

| 文件 | 操作 | 说明 |
|---|---|---|
| `src/views/ApiLog.vue` | 新增 | ElementPlus 重构小程序 apiLog 页面：<br>1. **工具栏**：`el-input` 搜索框（clearable）+ `el-button` 刷新 + `el-button` 清空（danger）<br>2. **连通性状态条**：自定义样式条展示 `unknown/checking/connected/failed`，含圆点颜色与探测耗时。失败时显示错误面板（标题 + 详情 + 「去设置」/「重试」按钮）<br>3. **后端地址条**：展示当前 baseURL，支持一键复制<br>4. **日志列表**：`el-card` 展示每条日志。头部含 METHOD 标签（GET 绿色/POST 橙色）、路径、状态码（>=400 红色）、时间、耗时（>1000ms 标红）、截断标签。点击展开/收起请求正文与响应数据（`pre` 标签展示 JSON）<br>5. **复制**：展开区域支持点击复制完整日志到剪贴板（`navigator.clipboard.writeText`）<br>6. **分页**：底部「加载更多」按钮，支持 `page` + `pageSize` |
| `src/utils/baseUrl.js` | 新增 | 提取小程序 `analyzeBaseUrl()` 与 `parseRequestError()` 逻辑：<br>- `analyzeBaseUrl(url)` → `{ valid, type, hint }`<br>- `parseRequestError(error, baseUrlInfo)` → `{ title, detail, code }` |
| `src/router/index.js` | 修改 | 新增路由 `{ path: 'api-log', name: 'ApiLog', component: () => import('@/views/ApiLog.vue'), meta: { title: '接口日志' } }` |
| `src/components/Layout.vue` | 修改 | `el-menu` 中新增菜单项「接口日志」，图标使用 `Document` |

**API 端点**：
- 连通性检测：`GET {baseURL}/api/v1/admin/health`，超时 4000ms
- 拉取日志：`GET {baseURL}/api/v1/admin/api-logs?page=&page_size=&search=`
- 清空日志：`POST {baseURL}/api/v1/admin/api-logs/clear`

**注意**：ApiLog.vue 内先通过 `getBaseUrl()` 工具拼接完整 URL 调用，待步骤四 axios 支持动态 baseURL 后可统一。

### 四、设置页面增强与全局配置体系

**目标**：新增云托管地址配置、关于信息、开发工具入口；让 axios 支持运行时 baseURL 切换。

| 文件 | 操作 | 说明 |
|---|---|---|
| `src/stores/app.js` | 修改 | 新增状态与方法：<br>- `cloudRunUrl`（ref）：从 `localStorage.getItem('cloudRunUrl')` 初始化，不存在则回退 `import.meta.env.VITE_API_BASE_URL`<br>- `setCloudRunUrl(url)`：校验 URL 有效性，保存到 localStorage，同步更新 axios 实例 baseURL<br>- `clearCloudRunUrl()`：移除 localStorage 配置，重置为环境变量默认值<br>- `getEffectiveBaseUrl()`：返回当前生效的后端地址 |
| `src/api/index.js` | 修改 | 支持动态 baseURL：在请求拦截器中读取 `localStorage.getItem('cloudRunUrl')`，若存在则覆盖 `config.baseURL`。保留现有 `ElMessage.error` 错误提示 |
| `src/views/Settings.vue` | 修改 | 全面重构，参考小程序 setting 页面分区：<br>1. **外观设置**：保留暗黑模式开关<br>2. **服务配置**：`el-card` 内放置云托管地址 `el-input`（clearable），输入验证提示，保存/清除按钮。保存时调用 `appStore.setCloudRunUrl()`<br>3. **名言管理入口**：可点击行，带右侧箭头，路由跳转 `/quote-manage`<br>4. **开发工具**：「接口日志」入口跳转 `/api-log`<br>5. **关于信息**：`el-descriptions` 展示应用名称（旺财百宝箱）、版本号、数据来源（AkShare）、开发框架（Vue3 + ElementPlus） |

**URL 校验逻辑**：
```js
function validateUrl(url) {
  if (!url.trim()) return '请输入云托管地址'
  try {
    const u = new URL(url)
    if (!['http:', 'https:'].includes(u.protocol)) return '请输入有效的 HTTP/HTTPS 地址'
  } catch {
    return '请输入有效的 URL 格式'
  }
  return ''
}
```

## 假设与决策

1. **字段统一**：名言字段从 `{ text, source }` 统一为 `{ text, author }`，与小程序保持一致。
2. **数据持久化**：名言与云托管地址统一使用 `localStorage`，通过 `src/utils/cache.js` 做读写封装。
3. **Pinia 模块外引用**：`src/api/index.js` 不在模块顶层引用 store，而是在请求拦截器中直接读取 `localStorage`，避免 Pinia 未初始化异常。
4. **剪贴板 API**：`navigator.clipboard` 在 localhost/HTTPS 环境下可用；若后续部署到 HTTP 环境需降级处理。
5. **暗黑模式**：新增页面中 ElementPlus 组件自动适配暗黑 CSS 变量；自定义卡片/标签颜色需手动在 `<style>` 中写 `:root.dark .class { ... }`。
6. **无新增 npm 依赖**：现有技术栈已足够支撑所有功能。

## 验证步骤

1. **名言管理**：访问「名言管理」页面，添加/编辑/删除名言，返回首页确认轮播内容已更新；刷新页面后数据持久化。
2. **恢复默认**：在名言管理页面点击「恢复默认」，确认列表恢复为初始数据。
3. **API 日志**：确保后端服务运行，访问「接口日志」页面，确认连通性检测通过，日志列表正常加载；测试搜索、加载更多、清空功能。
4. **云托管地址**：在设置页面修改云托管地址，保存后访问 API 日志页面，确认请求指向新地址；清除配置后恢复为默认。
5. **路由与菜单**：确认侧边栏菜单所有项点击后正确跳转，面包屑显示正常。
6. **暗黑模式**：切换暗黑模式后，新增页面（名言管理、API 日志）的自定义样式正确适配。
